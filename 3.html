Why is this red?

Now that you know what HTML is, let's talk about CSS, that is, Cascading Style Sheets

Simply put, CSS defines the presentation or style of your webpage. The style includes the fonts, color, padding, border, everything related to what the page should look like.

In other words, the HTML contains the content of the page, and the CSS describes the style.

Which is why the above heading is red! The CSS made it red!


Why is CSS separate from HTML?

This is a good question. In order to make a webpage one must write in TWO different languages, one in HTML, and the other in CSS.

Here is an example of what CSS looks like,

p { font-family: sans-serif; color: black; } .red { color: red; } .bold { font-weight: bold; }

Using the above CSS, I can write HTML like this,

<p>Is this <span class="red bold">red</span>?</p>

Which will look like this:

Is this red?


How easy!

In the old days of the World Wide Web we didn't have CSS, instead we mixed our content with our presentation.

We had things like <FONT> and we abused <TABLE> tags. Our HTML looked really ugly and was difficult to manage. Don't do this!

With CSS, we have a Separation of Concerns, this means that whoever is working on the content can do so indpendently of the presenation.

In practice, this means a web designer can focus on the CSS without having to worry about the content itself.


Let's get started with CSS

First things first, how do we link our HTML to our CSS?

Well, there are THREE ways to get your HTML and CSS to work together:

  1. External Stylesheet
  2. Internal Stylesheet
  3. Inline Style

Don't worry, it's much easier than it sounds. Let's go through these one by one.



The External Stylesheet

In this approach, you put all your CSS in a separate file. Easy!

This is the recommended approach, and you'll find most website do exactly this.

Let's try it out!


Step 1

Create a file called style.css in the same directory as example.html

Copy and paste the following into style.css and save.

body { font-family: sans-serif; color: gray; } .bold { font-weight: bold; } .purple { color: purple; } .red { color: red; } .gray, .grey { color: gray; }

Save this file.

Again, make sure it's called style.css and is in the same directory as example.html


Step 2

Now we just need to reference style.css in our HTML, it will look like this,

<link href="style.css" rel="stylesheet" type="text/css" />

Simple!

Here's an updated example.html, go and ahead and copy and paste this:

<!DOCTYPE html> <html> <head> <link href="style.css" rel="stylesheet" type="text/css" /> <meta charset="utf-8"> <title>My test page</title> </head> <body> Hello World! <p>You just created your first <span class="bold purple">webpage</span>, how exciting!</p> </body> </html>

Save the file, and now open your new example.html in a web browser.


If you did this correctly, what used to look like:

Hello World!

You just created your first webpage, how exciting!


Should now look like this:

Hello World!

You just created your first webpage, how exciting!


See how easy this is? You can modify the presentation without touching the HTML file.

And best of all, one stylesheet can be used for hundreds of HTML pages! CSS is great!

Now let's learn about the second way to get your CSS and HTML working together, the internal stylesheet.



The Internal Stylesheet

In this approach, you write your CSS directly inside the <head> element of your HTML file.

Here's a modified example.html, notice the new <style> element,

<!DOCTYPE html> <html> <head> <link href="style.css" rel="stylesheet" type="text/css" /> <meta charset="utf-8"> <title>My test page</title> <style> p { color: black; } </style> </head> <body> Hello World! <p>You just created your first <span class="bold purple">webpage</span>, how exciting!</p> </body> </html>

All we did here is put CSS directly inside of our HTML file, i.e.,

<style> p { color: black; } </style>

This starts to look ugly, the HTML and CSS are now sharing the same file.

If you have a simple webpage, this approach may work for you. I won't judge!

But if you want to mix it up, let's look at the third way to get CSS and HTML working together, the inline style.



Inline Style

An inline style allows you to write CSS directly inside each HTML element.

For example,

<p>I want this to be <span style="font-weight: bold; color: purple;">different</span></p>

I want this to be different



Cascading styles...

What does it mean that a style is cascading?

Let's look at what happens when you have multiple conflicting styles, such as different text color.

This should be gray, <span class="purple">inside here should be purple, <span class="red">this is red</span>, but is this <span class="bold red" style="color: black;">red or black or purple or gray</span>?</span>

Here's what that will look like:

This should be gray, inside here should be purple, this is red, but is this red or black or purple or gray?

The last part of that line has four competing color definitions, in order:

  1. the external stylesheet says it should be gray (defined in style.css),
  2. the internal stylesheet says it should be purple,
  3. the immediate tag says it should be bold red,
  4. and finally the inline style (style="color: black;") says it should be black.

It turns out to be bold black

The inline style will always take priority over any other inherited style.

The style definitions cascade, overwriting each other based on inheritance.

This is why CSS stands for Cascading Style Sheet.


Go ahead and experiment in example.html and style.css until you get the hang out of it.
