Let's break this down

<!DOCTYPE html>

This let's your web browser know what kind of document this is. The file extension also let's the browser know that this is an html document, so this line is rather silly and in many cases unecessary. People keep it around for older browsers, and out of tradition.


<html> <head> <meta charset="utf-8"> <title>My test page</title> </head>

This is the start of the HTML itself, everything between <html> and </html> is going to be written in html, that is, as Hyper-Text Markup Language.

and likewise, the <head> ... </head> tag will contain information about the document itself, such as,

<meta charset="utf-8">, this informs the browser that the character set is UTF-8, which means I can write things in 漢語 or Español or 한국어 or even العربية

The <title> tag is printed in the browser window.

Go ahead and reopen example.html in a different browser window and see for yourself, change the title to something else.


<body> Hello World! <p>You just created your first webpage, how exciting!</p> <p>To continue, click <a href="http://avant.net/lessons/2.html">here</a>.</p> </body> </html>

Here we have the <body> tag, and the end </html> tag, which ends the page. The body contains all the content that you want to see on the page.

The <p> tag is a paragraph, and is used to break up each paragraph of text.

The <a> tag is an anchor, and is used to link to a different website. These <a> links are what connects everything in the world-wide-web, and is why HTML is a hyper-text markup language.
